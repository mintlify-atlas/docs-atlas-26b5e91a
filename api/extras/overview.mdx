---
title: 'Extras Overview'
description: 'Manage additional items and services available in the system'
---

The Extras API allows you to manage additional items and services that can be added to bookings, such as chairs, tables, tablecloths, and other equipment. These endpoints require authentication via cookie-based JWT tokens.

## Available Endpoints

  <Card title="Create Extra" icon="plus" href="/api/extras/create">

  </Card>
  <Card title="List Extras" icon="list" href="/api/extras/list">

  <Card title="Get Extra" icon="file" href="/api/extras/get">

  </Card>
  <Card title="Update Extra" icon="pen" href="/api/extras/update">

  <Card title="Delete Extra" icon="trash" href="/api/extras/delete">

## Authentication

All extras endpoints require authentication using a JWT token stored in the `access_token` cookie. The user must have appropriate role permissions to access these endpoints.

## Common Fields

<ResponseField name="name" type="string">

<ResponseField name="value_add" type="number">
