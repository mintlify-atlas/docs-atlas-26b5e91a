---
title: 'Get Occupied Dates'
api: 'POST /space/occupied'
description: 'Retrieve all reservations for a specific space'
---

Returns all existing reservations for a specific space, including the reservation ID, start date, and end date. This is useful for checking space availability and displaying occupied time slots in a calendar or booking interface.

## Request Body

<ParamField body="v_space" type="string" required>
  Name of the space to query
  
  Example: `"SALÓN SOCIAL"`
</ParamField>

## Response

<ResponseField name="result" type="array">
  Array of reservation objects for the requested space
  
  <Expandable title="Reservation Object">
    <ResponseField name="ID_RESERVATION" type="string">
      Unique identifier of the reservation
      
      Example: `"RE001"`
    </ResponseField>
    
    <ResponseField name="INIT_DATE" type="string">
      Reservation start date and time (ISO 8601 format)
      
      Example: `"2025-01-15T10:00:00.000Z"`
    </ResponseField>
    
    <ResponseField name="END_DATE" type="string">
      Reservation end date and time (ISO 8601 format)
      
      Example: `"2025-01-15T18:00:00.000Z"`
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X POST https://api.example.com/space/occupied \
  -H "Content-Type: application/json" \
  -d '{
    "v_space": "SALÓN SOCIAL"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.example.com/space/occupied', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    v_space: 'SALÓN SOCIAL'
  })
});

const data = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.example.com/space/occupied',
    json={'v_space': 'SALÓN SOCIAL'}
)

data = response.json()
```

</CodeGroup>

## Use Cases

- Display a calendar showing occupied dates for a space
- Check space availability before creating a reservation
- Prevent double-booking by validating against existing reservations
- Build a booking interface with real-time availability

<Note>
  This endpoint does not require authentication, making it suitable for public-facing booking interfaces where users need to check availability before registering.
</Note>

<Info>
  The space name must match exactly (case-sensitive) with the registered space name in the system.
</Info>
