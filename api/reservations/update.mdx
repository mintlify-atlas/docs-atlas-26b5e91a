---
title: 'Update Reservation'
api: 'PUT /reserve/update'
description: 'Update an existing reservation'
---

This endpoint allows you to update the information of an existing reservation. You can modify customer data, dates, pax, rate, values, and the list of extras.

The user must be authenticated, and the JWT token is automatically obtained from the `access_token` cookie.

## Request

<ParamField body="v_id_reservation" type="string" required>
  Reservation ID to update (max 10 characters)
  
  Example: `"RSV00123"`
</ParamField>

<ParamField body="v_email" type="string" required>
  Updated customer email address (must be valid email format)
  
  Example: `"cliente_actualizado@example.com"`
</ParamField>

<ParamField body="v_phone_number" type="string" required>
  Updated customer phone number (7-13 characters)
  
  Example: `"3119876543"`
</ParamField>

<ParamField body="v_init_date" type="string" required>
  Updated reservation start date and time (ISO 8601 format)
  
  Example: `"2025-07-10T09:00:00Z"`
</ParamField>

<ParamField body="v_end_date" type="string" required>
  Updated reservation end date and time (ISO 8601 format, must be greater than v_init_date)
  
  Example: `"2025-07-10T14:00:00Z"`
</ParamField>

<ParamField body="v_pax" type="number" required>
  Updated number of people (must be positive integer)
  
  Example: `25`
</ParamField>

<ParamField body="v_status" type="string" required>
  Updated reservation status. Must be one of: `"EN PROGRESO"`, `"FINALIZADO"`
  
  Example: `"EN PROGRESO"`
</ParamField>

<ParamField body="v_extras" type="string" required>
  Updated JSON string containing selected extras. Must be sent as a string, not an object.
  
  Valid formats:
  - `"[{}]"` (empty extras)
  - `"[{\"id_extra\":1, \"quantity\":1, \"value_add\":3000}]"` (with extras)
  
  Example: `"[{\"id_extra\":1, \"quantity\":1, \"value_add\":3000}]"`
</ParamField>

<ParamField body="v_amount" type="number" required>
  Updated base amount (must be non-negative)
  
  Example: `95000`
</ParamField>

<ParamField body="v_total_value" type="number" required>
  Updated total value including extras (must be non-negative)
  
  Example: `100000`
</ParamField>

<ParamField body="v_fk_rate" type="number" required>
  Updated rate ID associated with the reservation (must be positive integer)
  
  Example: `4`
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseExample>
```json Response
{
  "message": "Update Exitoso"
}
```
</ResponseExample>

## Error Responses

<ResponseField name="400" type="Bad Request">
  Validation error in the sent data. Common issues:
  - Invalid email format
  - End date is not greater than start date
  - Missing required fields
  - Invalid data types or constraints
</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Token not sent or invalid. Ensure you're authenticated and the `access_token` cookie is present.
</ResponseField>

<ResponseField name="404" type="Not Found">
  No reservation exists with the provided ID.
</ResponseField>

<ResponseField name="409" type="Conflict">
  The selected dates are occupied or not available. Try different dates.
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  Internal server error. Contact support if this persists.
</ResponseField>

## Notes

- The `v_id_reservation` field is required to identify which reservation to update
- The `v_name` field cannot be updated (it's not included in the update schema)
- The `v_extras` field must be sent as a JSON string, not as a JSON object or array
- The end date must be strictly greater than the start date
- The system validates date availability and will return a 409 error if the new dates are already occupied
- The authenticated user's employee ID is automatically associated with the update operation

## What Can't Be Updated

The following fields from the original reservation cannot be updated:
- `v_name` - Customer name remains unchanged

If you need to change the customer name, you may need to delete and recreate the reservation.
